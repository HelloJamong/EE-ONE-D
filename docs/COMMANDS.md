## 1. 관리자 설정 (Config)

봇의 기본 채널 설정 및 봇 상태를 관리합니다.

### 권한 요구사항
- 명령어 실행자 권한: `Administrator`

### 명령어

#### `/config set`
봇이 사용할 채널들을 설정합니다.

**옵션:**
- `role_channel` - 역할 선택 패널이 표시될 채널 (선택)
- `admin_channel` - 관리 명령어를 사용할 채널 (선택)
- `log_channel` - 감사 로그가 기록될 채널 (선택, **설정 시 자동 활성화**)
- `notification_channel` - 공지사항이 발송될 채널 (선택)
- `welcome_channel` - 웰컴 메시지가 표시될 채널 (선택)

**사용 예시:**
```
/config set admin_channel:#관리자-전용
/config set log_channel:#감사로그
/config set notification_channel:#공지사항
/config set welcome_channel:#환영
/config set role_channel:#역할선택
```

**참고사항:**
- 한 번에 여러 채널을 설정할 수 있습니다.
- 설정하지 않은 옵션은 기존 값이 유지됩니다.
- `admin_channel` 설정 시 이후 모든 관리 명령어는 해당 채널에서만 사용 가능합니다.
- `log_channel` 설정 시 감사 로그가 자동으로 활성화됩니다.

#### `/config bot_status`
봇의 상태 메시지를 설정합니다.

**옵션:**
- `type` - 활동 타입 선택 (필수)
  - `플레이중`: "~~ 플레이중"으로 표시
  - `시청중`: "~~ 시청중"으로 표시
  - `듣는중`: "~~ 듣는중"으로 표시
- `text` - 표시할 텍스트 (필수, 최대 128자)

**사용 예시:**
```
/config bot_status type:시청중 text:BIG BROTHER IS WATCHING YOU
/config bot_status type:플레이중 text:디스코드 관리
/config bot_status type:듣는중 text:Spotify
```

**참고사항:**
- DB에 저장되어 봇 재시작 시에도 유지됩니다.
- 실시간으로 봇 프로필에 반영됩니다.

#### `/config show`
현재 설정된 채널 정보를 확인합니다.

**사용 예시:**
```
/config show
```

**출력 예시:**
```
role_panel_channel_id: 123456789012345678
admin_config_channel_id: 234567890123456789
log_channel_id: 345678901234567890
notification_channel_id: 456789012345678901
welcome_channel_id: 567890123456789012
activity_type: WATCHING
activity_text: BIG BROTHER IS WATCHING YOU
updated_at: 2026-03-13T10:00:00.000Z
```

## 2. 역할 선택 패널 (Role Panels)

사용자가 버튼을 클릭하여 역할을 선택/해제할 수 있는 패널을 관리합니다.

### 권한 요구사항
- 명령어 실행자 권한: `Administrator`
- `admin_config_channel`에서만 사용 가능 (설정된 경우)

### 명령어

#### `/panel create`
새로운 역할 선택 패널을 생성합니다.

**옵션:**
- `mode` - 패널 모드 선택 (필수)
  - `MULTI`: 여러 역할 동시 선택 가능
  - `SINGLE`: 한 번에 하나의 역할만 선택 가능

**사용 예시:**
```
/panel create mode:MULTI
/panel create mode:SINGLE
```

**참고사항:**
- 패널 ID가 자동으로 생성됩니다.
- 생성 후 `/panel add`로 역할을 추가해야 합니다.

#### `/panel add`
패널에 역할을 추가합니다.

**옵션:**
- `panel_id` - 패널 ID (필수)
- `role` - 추가할 역할 (필수)
- `label` - 버튼에 표시될 텍스트 (필수)
- `emoji` - 버튼에 표시될 이모지 (선택)
- `description` - 역할 설명 (선택, `:emoji_name:` 형식 사용 가능)

**사용 예시:**
```
/panel add panel_id:abc123 role:@게임 label:게임 알림
/panel add panel_id:abc123 role:@공지 label:공지 알림 emoji:📢
/panel add panel_id:abc123 role:@이벤트 label:이벤트 description::tada: 이벤트 알림을 받습니다
```

**커스텀 이모지 자동 변환:**
- Description에 `:emoji_name:` 형식으로 입력하면 서버의 커스텀 이모지로 자동 변환됩니다.
- 예: `:custom_emoji:` → `<:custom_emoji:123456789>`
- 애니메이션 이모지도 지원합니다.

#### `/panel remove`
패널에서 역할을 제거합니다.

**옵션:**
- `panel_id` - 패널 ID (필수)
- `role` - 제거할 역할 (필수)

**사용 예시:**
```
/panel remove panel_id:abc123 role:@게임
```

#### `/panel list`
생성된 모든 패널 목록을 확인합니다.

**사용 예시:**
```
/panel list
```

**출력 예시:**
```
패널 목록:
- ID: abc123, 모드: MULTI, 항목 수: 3, 메시지 ID: 987654321098765432
- ID: def456, 모드: SINGLE, 항목 수: 2, 메시지 ID: -
```

#### `/panel set_message`
패널의 안내 메시지를 설정합니다.

**옵션:**
- `panel_id` - 패널 ID (필수)
- `message` - 안내 메시지 내용 (필수)

**사용 예시:**
```
/panel set_message panel_id:abc123 message:원하는 역할을 선택하세요!
```

#### `/panel publish`
패널을 역할 선택 채널에 게시합니다.

**옵션:**
- `panel_id` - 패널 ID (필수)

**사용 예시:**
```
/panel publish panel_id:abc123
```

**참고사항:**
- `/config set role_channel`로 채널을 먼저 설정해야 합니다.
- 패널에 최소 1개 이상의 역할이 추가되어 있어야 합니다.
- 게시된 패널은 자동으로 업데이트됩니다.

## 3. 커스텀 명령어 (Custom Commands)

관리자가 직접 슬래시 명령어를 생성/관리할 수 있습니다.

### 권한 요구사항
- 명령어 실행자 권한: `Administrator`
- `admin_config_channel`에서만 사용 가능 (설정된 경우)

### 명령어

#### `/cmd add`
새로운 커스텀 명령어를 추가합니다.

**사용 예시:**
```
/cmd add name:rules
```

**사용 방법:**
1. 명령어 실행 시 Modal(입력창)이 표시됩니다.
2. **명령어 이름**: 슬래시 명령어 이름 입력 (영문, 소문자, 숫자, 하이픈만 가능)
3. **미리보기 설명** (선택): Discord 슬래시 명령어 목록에 표시될 설명 (최대 100자)
4. **응답 내용**: 명령어 실행 시 응답할 내용 (최대 4000자)

**고급 기능:**

1. **줄바꿈 사용** - Enter 키로 자연스럽게 줄바꿈을 입력할 수 있습니다.
   ```
   안녕하세요!
   오늘도 좋은 하루 되세요!
   ```

2. **랜덤 응답** - `|||`로 구분하여 여러 응답 중 하나를 랜덤으로 출력합니다.
   ```
   1|||2|||3|||4|||5|||6
   안녕하세요!|||반갑습니다!|||어서오세요!|||환영합니다!
   ```

3. **임베드 형식** - `EMBED:`로 시작하고 `|||`로 제목과 내용을 구분합니다.
   ```
   EMBED:서버 점검 안내|||오늘 밤 12시에 서버 점검이 있습니다.
   약 1시간 소요될 예정입니다.
   ```

4. **랜덤 + 임베드 조합** - 여러 임베드 중 랜덤으로 선택합니다.
   ```
   EMBED:명언 1|||노력은 배신하지 않는다|||EMBED:명언 2|||오늘 할 일을 내일로 미루지 말라
   ```

5. **이미지 URL** - 여러 이미지 URL을 포함할 수 있습니다.
   ```
   https://example.com/image1.png
   https://example.com/image2.png
   ```

**명령어 이름 규칙:**
- **한글 사용 불가**: Discord 슬래시 명령어는 한글을 지원하지 않습니다.
- **영문 소문자**: 대문자는 자동으로 소문자로 변환됩니다.
- **숫자**: 0-9 사용 가능
- **하이픈**: 단어 구분용으로 사용 가능 (예: `server-info`)
- **길이**: 1-32자

**사용 가능한 예시:**
- `rules` ✅
- `server-info` ✅
- `help2` ✅
- `규칙` ❌ (한글 불가)
- `Server-Info` ❌ (대문자 불가)
- `server_info` ❌ (언더스코어 불가)

**참고사항:**
- 예약어 (`panel`, `config`, `cmd`, `role`, `noti`, `welcome`, `help`, `version`)는 사용할 수 없습니다.
- 추가된 명령어는 모든 채널에서 사용 가능합니다.
- 봇 재시작 없이 즉시 등록됩니다.
- `|||`를 응답 내용으로 사용할 수 없습니다 (구분자로 사용됨).

#### `/cmd edit`
기존 커스텀 명령어를 수정합니다.

**옵션:**
- `name` - 수정할 명령어 이름 (필수)

**사용 예시:**
```
/cmd edit name:rules
```

**사용 방법:**
1. 명령어 실행 시 Modal에 기존 값이 자동으로 채워져 표시됩니다.
2. 수정할 내용을 입력하고 제출합니다.

#### `/cmd remove`
커스텀 명령어를 삭제합니다.

**옵션:**
- `name` - 삭제할 명령어 이름 (필수)

**사용 예시:**
```
/cmd remove name:rules
```

#### `/cmd list`
등록된 모든 커스텀 명령어를 확인합니다.

**사용 예시:**
```
/cmd list
```

**출력 예시:**
```
커스텀 명령어 목록:
- /rules: 서버 규칙을 확인하려면 #규칙 채널을 참고하세요.
- /discord-link: 디스코드 서버 주소: https://discord.gg/example
```

#### `/cmd reload`
커스텀 명령어를 Discord API에 재등록합니다.

**사용 예시:**
```
/cmd reload
```

**참고사항:**
- 명령어가 정상적으로 표시되지 않을 때 사용합니다.
- 일반적으로는 자동으로 등록되므로 사용할 필요가 없습니다.

## 4. 공지사항 관리 (Notifications)

관리자가 봇을 통해 공지사항을 작성, 수정, 삭제할 수 있습니다.

### 권한 요구사항
- 명령어 실행자 권한: `Administrator`
- `admin_config_channel`에서만 사용 가능 (설정된 경우)

### 명령어

#### `/noti send`
새로운 공지사항을 작성하고 발송합니다.

**사용 예시:**
```
/noti send
```

**사용 방법:**
1. 명령어 실행 시 Modal(입력창)이 표시됩니다.
2. **제목**: 공지사항 제목을 입력합니다 (최대 256자).
3. **내용**: 공지사항 내용을 입력합니다 (최대 4000자).
4. 제출하면 설정된 공지사항 채널에 Embed 형식으로 발송됩니다.

**참고사항:**
- `/config set notification_channel`로 채널을 먼저 설정해야 합니다.
- 내용에 @역할, #채널, URL, 이미지 링크 등을 포함할 수 있습니다.
- 발송된 메시지 ID가 표시되며, 수정/삭제 시 필요합니다.

#### `/noti edit`
이미 발송된 공지사항을 수정합니다.

**옵션:**
- `message_id` - 수정할 메시지 ID (필수)

**사용 예시:**
```
/noti edit message_id:1234567890123456789
```

**사용 방법:**
1. 메시지 ID를 입력하여 명령어를 실행합니다.
   - 메시지 우클릭 → "메시지 ID 복사"로 ID를 얻을 수 있습니다.
2. Modal에 기존 제목과 내용이 자동으로 채워져 표시됩니다.
3. 수정할 내용을 입력하고 제출합니다.

**참고사항:**
- 메시지 ID는 숫자만 입력해야 합니다.
- 개발자 모드를 활성화해야 메시지 ID를 복사할 수 있습니다.

#### `/noti remove`
공지사항을 삭제합니다.

**옵션:**
- `message_id` - 삭제할 메시지 ID (필수)

**사용 예시:**
```
/noti remove message_id:1234567890123456789
```

**참고사항:**
- 메시지가 완전히 삭제되며 복구할 수 없습니다.
- 감사 로그에 삭제 기록이 남습니다.

#### `/noti poll`
투표를 생성합니다.

**옵션:**
- `duration` - 투표 마감 시간 (시간 단위, 기본값: 1, 최대: 24) (선택)
- `allow_multiple` - 중복 투표 허용 여부 (기본값: false) (선택)

**사용 예시:**
```
/noti poll
/noti poll duration:3
/noti poll duration:12 allow_multiple:true
```

**사용 방법:**
1. 명령어 실행 시 Modal(입력창)이 표시됩니다.
2. **투표 제목**: 투표의 제목을 입력합니다 (최대 256자).
3. **투표 항목**: 항목을 줄바꿈으로 구분하여 입력합니다 (최소 2개, 최대 10개).
4. 제출하면 설정된 공지사항 채널에 버튼 형식으로 투표가 생성됩니다.

**투표 참여 방법:**
- 숫자 버튼(1️⃣~🔟)을 클릭하여 투표합니다.
- 같은 버튼을 다시 클릭하면 투표가 취소됩니다.
- 중복 투표 불가 모드에서는 다른 항목 클릭 시 기존 표가 자동으로 교체됩니다.

**참고사항:**
- `/config set notification_channel`로 채널을 먼저 설정해야 합니다.
- 마감 시간이 지나면 버튼이 비활성화되고 최종 결과가 표시됩니다.
- 봇 재시작 시에도 진행 중인 투표의 마감 타이머가 자동으로 복구됩니다.
- 감사 로그에 투표 생성 기록이 남습니다.

## 5. 웰컴 메시지 (Welcome)

신규 멤버가 서버에 입장할 때 표시되는 환영 메시지와 역할 부여 시스템을 관리합니다.

### 권한 요구사항
- 명령어 실행자 권한: `Administrator`
- `admin_config_channel`에서만 사용 가능 (설정된 경우)

### 명령어

#### `/welcome setup`
웰컴 메시지를 설정합니다.

**옵션:**
- `role1` - 부여할 역할 1 (필수)
- `role2` - 부여할 역할 2 (선택)
- `role3` - 부여할 역할 3 (선택)
- `role4` - 부여할 역할 4 (선택)
- `role5` - 부여할 역할 5 (선택)

**사용 예시:**
```
/welcome setup role1:@멤버
/welcome setup role1:@멤버 role2:@공지 role3:@이벤트
```

**사용 방법:**
1. 역할을 선택하고 명령어를 실행합니다 (최대 5개).
2. Modal(입력창)이 표시됩니다.
3. **제목**: 웰컴 메시지 제목 (최대 256자)
4. **내용**: 웰컴 메시지 내용 (최대 4000자)
5. **버튼 이모지** (선택): 버튼에 표시될 이모지
6. **버튼 레이블**: 버튼에 표시될 텍스트
7. 제출하면 웰컴 채널에 메시지가 게시됩니다.

**참고사항:**
- `/config set welcome_channel`로 채널을 먼저 설정해야 합니다.
- 버튼 클릭 시 선택한 모든 역할이 동시에 부여됩니다.
- 이미 모든 역할을 보유한 경우 "이미 인증되었습니다" 메시지가 표시됩니다.
- 길드당 1개의 웰컴 메시지만 지원합니다.

#### `/welcome edit`
기존 웰컴 메시지를 수정합니다.

**사용 예시:**
```
/welcome edit
```

**사용 방법:**
1. 명령어 실행 시 Modal에 현재 설정값이 자동으로 채워져 표시됩니다.
2. 수정할 내용을 입력하고 제출합니다.
3. **역할 ID**: 쉼표로 구분하여 입력 (예: `123456789,987654321`)

#### `/welcome remove`
웰컴 메시지를 삭제합니다.

**사용 예시:**
```
/welcome remove
```

**참고사항:**
- 메시지와 DB 레코드가 모두 삭제됩니다.
- 복구할 수 없으므로 주의하세요.

## 7. 감사 로그 (Audit)

서버에서 발생하는 주요 이벤트를 자동으로 기록합니다.

### 설정 방법

감사 로그는 `/config set log_channel`로 로그 채널을 설정하면 **자동으로 활성화**됩니다.

```
/config set log_channel:#감사로그
```

**설정 완료 메시지:**
```
설정을 갱신했습니다.
...
log_channel_id: 123456789012345678

✅ 감사 로그가 자동으로 활성화되었습니다.
```

**참고:**
- 로그 채널을 설정하는 것만으로 감사 로그가 즉시 활성화됩니다.
- 별도의 활성화 명령어가 필요 없습니다.
- `/config show`로 현재 상태를 확인할 수 있습니다.

### 자동 기록 이벤트

#### 음성 채널 이벤트
- **입장**: 사용자가 음성 채널에 입장할 때
- **퇴장**: 사용자가 음성 채널에서 퇴장할 때
- **이동**: 사용자가 음성 채널 간 이동할 때 (v1.0.3+)

#### 메시지 이벤트
- **삭제**: 메시지가 삭제될 때 (원본 내용 및 이미지 기록)
- **수정**: 메시지가 수정될 때 (수정 전/후 내용 기록)

#### 멤버 이벤트
- **서버 입장**: 새로운 멤버가 서버에 입장할 때
- **서버 퇴장**: 멤버가 서버에서 퇴장할 때
- **역할 변경**: 멤버의 역할이 추가/제거될 때

#### 설정 변경 이벤트
- **채널 설정**: 봇 채널 설정이 변경될 때
- **패널 생성/수정**: 역할 패널이 생성되거나 수정될 때
- **커스텀 명령어**: 커스텀 명령어가 추가/삭제될 때
- **공지사항**: 공지사항이 발송/수정/삭제될 때

### 로그 형식 (DynoBot 스타일)

- **Author**: 유저 프로필 사진과 이름 표시
- **Description**: 간결한 액션 중심 설명
- **Footer**: 고유 ID 정보 표시
  - 음성/멤버 로그: `ID: <유저ID>`
  - 메시지 로그: `User ID: <유저ID> | Message ID: <메시지ID>`

**로그 메시지 예시:**
```
[음성 입장]
joined voice channel #일반-음성

[메시지 삭제]
메시지 내용...
(이미지가 있는 경우 임베드에 포함)
Jump to Message: [링크]

[역할 부여]
was granted role @게임
```

### 특수 기능

#### 이미지 영구 보관
- 메시지 삭제 시 첨부된 이미지를 자동으로 다운로드하여 로그 채널에 재업로드합니다.
- Discord CDN URL 만료 문제를 해결하여 영구 보관됩니다.
- 여러 이미지 첨부 지원

### 로그 저장 방식

#### 1. 데이터베이스 (항상 기록)
모든 감사 로그는 `audit_events` 테이블에 자동으로 저장됩니다.

**저장 필드:**
- `event_type`: 이벤트 종류
- `actor_id`: 작업을 수행한 사용자 ID
- `target_id`: 대상 사용자/메시지 ID
- `channel_id`: 이벤트가 발생한 채널 ID
- `details`: 추가 세부 정보 (JSON)
- `created_at`: 발생 시각

#### 2. 로그 채널 (설정 시 기록)
`log_channel`이 설정되어 있으면 주요 이벤트가 해당 채널에도 임베드 형식으로 표시됩니다.

### 감사 로그 비활성화

감사 로그를 비활성화하려면 로그 채널 설정을 제거하면 됩니다.

**방법:**
1. DB에서 직접 `log_channel_id`를 `NULL`로 설정
2. 또는 다른 채널로 변경하여 해당 채널에서만 로그 확인

## 10. 자동 기능

사용자 명령 없이 자동으로 작동하는 기능들입니다.

### 커스텀 이모지 확대

**기능:**
- 커스텀 이모지만 포함된 메시지를 자동으로 확대하여 표시합니다.

**작동 조건:**
- 메시지에 커스텀 이모지가 단독으로 포함된 경우
- 텍스트나 다른 내용이 없어야 합니다.

**예시:**
```
입력: :custom_emoji:
출력: [확대된 이모지 이미지]
```

**참고사항:**
- 서버별 니트로 프로필 이미지가 우선 사용됩니다.

### 디시인사이드 미리보기

**기능:**
- 디시인사이드 게시글 링크를 Embed 형식으로 미리보기를 제공합니다.

**작동 조건:**
- 메시지에 디시인사이드 URL이 단독으로 포함된 경우
- 갤러리 게시글 URL만 지원합니다.

**지원 URL 형식:**
- 데스크톱:
  - `https://gall.dcinside.com/board/view/?id=...&no=...`
  - `https://gall.dcinside.com/mgallery/board/view/?id=...&no=...`
- 모바일:
  - `https://m.dcinside.com/board/{갤러리ID}/{게시글번호}`
  - `https://m.dcinside.com/board/{갤러리ID}/{게시글번호}?recommend=1`
  - 쿼리 파라미터 포함 URL 지원 (v1.0.9+)

**미리보기 내용:**
- 게시글 제목
- 갤러리명
- 본문 요약 (최대 300자, 있는 경우만)
- 작성자 정보 (메시지 작성자)

**참고사항:**
- 캐싱이 적용되어 동일한 URL은 빠르게 로드됩니다 (60초 TTL).
- 접근이 제한된 게시글은 미리보기가 제공되지 않습니다.
- 모바일 URL은 자동으로 데스크톱 URL로 변환됩니다.
- 서버별 니트로 프로필 이미지가 우선 사용됩니다.

## 부록

### 개발자 모드 활성화

메시지 ID를 복사하려면 개발자 모드를 활성화해야 합니다.

1. 디스코드 설정 열기 (톱니바퀴 아이콘)
2. 좌측 메뉴에서 "고급" 선택
3. "개발자 모드" 토글 활성화
4. 메시지 우클릭 → "메시지 ID 복사" 옵션이 표시됩니다.

### 권한 확인

#### 사용자(본인) 권한 확인

관리 명령어를 사용하려면 **본인이** `Administrator` 권한을 가지고 있어야 합니다.

본인의 `Administrator` 권한 확인 방법:

1. 서버 설정 → 역할
2. 본인의 역할 선택
3. "권한" 탭에서 "관리자" 권한이 활성화되어 있는지 확인

#### 봇 권한

봇 자체도 다음 권한이 필요합니다 (봇 역할에 설정):
- `Manage Roles` (역할 관리)
- `Read Messages/View Channels` (메시지 읽기/채널 보기)
- `Send Messages` (메시지 전송)
- `Manage Messages` (메시지 관리)
- `Embed Links` (링크 임베드)

**참고:** 명령어 실행 권한과 봇 권한은 별개입니다. 사용자 본인이 Administrator 권한이 있어야 관리 명령어를 실행할 수 있습니다.

### 채널 ID vs 메시지 ID

- **채널 ID**: 채널 우클릭 → "채널 ID 복사"
- **메시지 ID**: 메시지 우클릭 → "메시지 ID 복사"
- 두 ID는 서로 다르므로 주의하세요.

### 문제 해결

#### 명령어가 표시되지 않아요
- 봇이 온라인 상태인지 확인하세요.
- `/cmd reload`를 실행하여 명령어를 재등록하세요.
- 디스코드를 재시작하세요.

#### "이 명령어는 지정된 관리자 채널에서만 사용할 수 있습니다" 오류
- `/config show`로 현재 설정을 확인하세요.
- `admin_config_channel_id`에 설정된 채널에서만 관리 명령어를 사용할 수 있습니다.

#### "공지사항 채널이 설정되지 않았습니다" 오류
- `/config set notification_channel:#채널명`으로 채널을 먼저 설정하세요.

#### "웰컴 채널이 설정되지 않았습니다" 오류
- `/config set welcome_channel:#채널명`으로 채널을 먼저 설정하세요.

#### 패널을 게시할 수 없어요
- `/config set role_channel:#채널명`으로 역할 채널을 먼저 설정하세요.
- 패널에 최소 1개 이상의 역할이 추가되어 있는지 확인하세요.

#### "/welcome setup 역할 5개 선택 시 오류"
- v1.0.8부터 수정되었습니다. 최신 버전으로 업데이트하세요.

#### "디시인사이드 링크가 임베드로 표시되지 않아요"
- v1.0.10부터 빈 description 오류가 수정되었습니다.
- 모바일 URL 쿼리 파라미터는 v1.0.9부터 지원됩니다.
- 최신 버전으로 업데이트하세요.
